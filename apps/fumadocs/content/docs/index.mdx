## What GitPal does

- Review pull requests with repository context, related work, file history, and selected tools.
- Assign labels from the repository's existing label catalog for issues and pull requests.
- Request provider-native reviewers from connected GitHub or GitLab identities.
- Route model calls through GitPal-managed providers or a user's own keys.
- Persist generation usage and settlement details for auditability and billing.

## How to think about the app

- `AI review`: main reviewer, walkthrough generation, fun extras, and label suggestions.
- `Integrations`: GitHub, GitLab, webhook delivery, and provider-native actions.
- `Tools`: built-in tools, MCP bindings, repository overrides, and search context.
- `Billing`: wallet debits, BYOK routing, and generation cost tracking.
- `Operations`: production safety, logs, retries, and troubleshooting.

Use the category pages to configure the product end to end, then use the settings preview to verify the comment that GitPal will publish on real PRs.
