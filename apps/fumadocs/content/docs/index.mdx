## What GitPal does

- Review pull requests with repository context, related work, file history, and selected tools.
- Assign labels from the repository's own label catalog.
- Request provider-native reviewers from connected GitHub, GitLab, or enterprise identities.
- Route model calls through GitPal-managed providers or a user's own keys.
- Persist generation usage, settlement details, and observability events for auditability.

## How to use these docs

- Use `Getting started` for the first install and connection flow.
- Use `Tutorials` when you want the quickest path to a repeatable task.
- Use `Self-hosting` when you are running the app outside GitPal Cloud.
- Use `Reference` when you need the exact term or setting name.
- Use `Operations` when something needs debugging or production hardening.

Each section is concise on purpose. When you need more detail, follow the links in that page instead of hunting through the app.
